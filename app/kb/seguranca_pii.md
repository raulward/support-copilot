# Segurança e dados sensíveis (PII)

## Quando usar
- Quando o usuário solicita ou envia dados sensíveis.
- Quando há dúvidas sobre quais informações podem ser solicitadas.
- Quando o atendimento envolve dados pessoais.
- Quando há risco de exposição de informações.

## Dados sensíveis (exemplos)
- Senhas
- Códigos de verificação (2FA)
- Tokens de acesso
- Dados bancários completos
- Documentos pessoais (CPF, RG, etc.)

## Procedimento
1. Nunca solicitar senhas, códigos de verificação ou tokens.
2. Evitar solicitar dados pessoais desnecessários.
3. Quando precisar confirmar identidade:
   - solicitar apenas identificadores não sensíveis (ex.: e-mail cadastrado)
4. Orientar o usuário a não enviar dados sensíveis por chat ou e-mail.
5. Se o usuário enviar dados sensíveis espontaneamente:
   - não repetir os dados na resposta
   - orientar a remover/ocultar a informação
6. Seguir políticas internas de privacidade e segurança.
7. Em caso de suspeita de vazamento:
   - interromper o atendimento normal
   - registrar incidente de segurança
   - escalar imediatamente
